3. Flujo de trabajo en Git
- Cada integrante clonó el repositorio en su equipo.
- Se creó una rama con el comando:
      git checkout -b cambios1
- Los cambios se subieron con:
      git add .
      git commit -m "mensaje"
      git push origin cambios1
- Se hizo uso de pull y rebase para unir versiones.
